# 数据类型

Redis中的数据类型与Java中的数据类型

string String
hash HashMap
list LinkedList
set HashSet
sorted_set TreeSet

说明：

  1. Redis本真是一个Map，其中所有的数据都是采用可以key：value的形式来存储。

  2. Redis中的数据类型指的是存储的数据的类型，也就是Value部分，Redis中的Key全部是字符串。

# 一、string

# 1.1 基本操作

  1. 添加、修改数据

    set key value

  2. 获取数据

    get key

  3. 删除数据

    del key

  4. 添加、修改多个数据

    mset key1 value1 key2 value2

  5. 获取多个数据

    mget key1 key2

  6. 获取数据长度

    strlen key

  7. 追加信息到原始信息候后部（如果原始信息存在就追加，否则新建）

    append key value

# 1.2 单数据操作与多数据操作的选择

  1. 在数据量非常小的时候，单数据操作与多数据操作没有太大区别
  2. 在数据量比较大时，可以选择将数据切割后，采用多数据操作

# 1.3 扩展操作

  1. 增加值

    incr key
incrby key increment
incrbyfloat key increment

  2. 减少值

    decr key
decrby key increment

  3. 设置数据的生命周期

    setex key seconds value
psetex key milliseconds value

  4. 数据操作的注意事项

    • 表示运行结果是否成功（返回0：失败，返回1：成功）

    • 表示运行结果，返回的数字代表执行成功的个数

    • 未获取到数据，返回nil，相当于null

    • Redis中数据最大存储量512M

    • 数值的最大范围是：Java中long的范围

    • Key的设置：表明:主键名:主键值:字段名

  5. 举例

    set user:id:123:fans 12345
set user:id:123:blogs 789
set user:id:12345 {id:12345, name:haha,fans:345,blogs:897}

  6. 说明

    • String在Redis内部存储默认就是一个字符串，当遇到增减类操作incr，decr时会转换成数值型进行计算。
    • Redis所有的操作都是原子性的，采用单线程处理所有业务，命令是一个一个执行的，因此无需考虑并发带来的数据安全问题。
    • 按数值进行操作的数据，如果原始数据不能转成数值，或超过了Redis数值的上限范围，会报错。
    • Redis可以用于控制数据库表主键Id，为数据库表主键提供生成策略，保障数据库表的主键唯一性。适用于所有的数据库，且支持数据库集群。

# 二、hash

redis中的hash可以用户抢购、限购、限量发放优惠券、激活码等业务的数据库设计。

  1. 新的存储需求：对一系列存储数据进行编组，方便管理，典型应用是存储对象信息
  2. 需求的内存结构：一个存储空间保存多个键值对数据
  3. hash类型：底层使用哈希表结构实现数据存储
  4. hash存储结构优化
    1. 如果field数量较少，存储结构优化为数组
    2. 如果field数量较多，存储结构使用HashMap

# 2.1 基本操作

  1. 添加、修改数据

    hset key field value

  2. 获取数据

    hget key field
hgetall key

  3. 删除数据

    hdel key field1 field2

  4. 添加、修改多个数据

    hmset key field1 value1 field2 value2

  5. 获取多个数据

    hmget key field1 field2

  6. 获取哈希表中字段的数量

    hlen key

  7. 判断哈希表中是否存在指定的字段

    hexists key field

# 2.2 扩展操作

  1. 获取所有的字段名或字段值

    hkeys key
hvals key

  2. 设置指定字段的数值数据增加指定范围的值

    hincrby key field increment
hincrbyfloat key field increment

  3. hsetnx用法 如果field对应的Value没有值，则设置值；如果已经有值，则不变

    hsetnx key field value

# 2.3 注意事项

  1. hash类型下的Value只能存储字符串，不允许存储其他数据类型，不存在嵌套现象。如果数据未获取到，对应的返回值是nil。

  2. 每个hash可以存储2^32-1个键值对。

  3. hash类型十分贴近对象的数据存储形式，并且可以灵活添加与删除对象属性。但hash设计初衷不是为了存储大量对象而设计的，不能将hash作为对象列表使用。

  4. hgetall操作可以获取全部属性，如果内部field过多，遍历整体数据效率会很低，有可能会成为数据访问的瓶颈。

# 2.4 举例

  1. 实现一个购物车

    业务分析：功能：添加、浏览、更改商品数量、删除、清空 解决方案：

    1. 以客户id作为key，每位客户创建一个hash存储对应的购物车信息
    2. 将商品编号作为field，购买数量作为value进行存储
    hmset 001 g01 100 g02 200
hmset 002 g01 200 g03 50
将描述信息也存储在hash中
hmset 003 g01:nums 12 g01:info {...}
    1. 添加商品：追加一个field与value对
    hset 005 g02:nums 56 g02:info {..} g03:nums 10 g03:info {...}
    1. 浏览：遍历hash
    hgetall 001
    1. 更改数量：自增、自减、设置value的值
    hincrby 005 g02:nums 10
hincrby 005 g03:nums -2
    1. 删除商品：删除field
    hdel 005 g02:info
    1. 清空：删除key
    hdel 005 g01:nums g01:info g02:nums g02:info

  2. 秒杀

    双十一，销售手机充值卡的商家对移动、联通和电信的30元、50元、100元商品推出抢购活动，每种商品抢购上限1000张。

    解决方案：

    • 以商家id作为key
    • 将参与秒杀的商品Id作为field
    • 将参与抢购的商品数量作为对应的value
    • 抢购时使用降值得防止控制产品的数量

    初始化

    hmset p01 c30 1000 c50 1000 c100 1000

    买家秒杀

    hincrby p01 c30 -1
hincrby p01 c100 -20

# 三、list

  • 数据存储需求：存储多个数据，并对数据进入存储空间的顺序进行区分
  • 需要的存储结构：一个存储空间保存多个数据，且通过数据可以体现进入顺序
  • list类型：保存多个数据，底层使用双向链表存储结构实现

# 3.1 基本操作

  1. 添加、修改数据
lpush key v1 v2 v3
rpush key v1 v2 v3
  1. 获取数据
lrange key start stop
lindex key index
llen key
  1. 获取并移除数据
lpop key
rpop key

# 3.2 扩展操作

  1. 在规定时间内获取并移除数据 从多个list中获取数据，如果当中任何一个list中有数据，就立即取出数据
blpop key1 key2  timeout
brpop key1 key2 timeout
  1. 移除指定数据
lrem key count value

count值元素的个数，因为list中的元素可以重复，Value是需要移除的元素

# 3.3 应用场景

  1. 在微信朋友圈，按照点赞顺序展示好友信息

    rpush k1 a b c d e
lrem k1 1 c

  2. 使用队列模型解决多路信息汇总合并的问题

  3. 使用栈模型解决最新消息的问题

# 3.4 注意事项

  1. list中保存的数据都是String类型的，数据总量有限
  2. list具有索引的概念，但是操作数据时通常以队列的形式进行入队出队，或以栈的形式进行入栈与出栈
  3. 获取全部数据后，索引设置为-1（最后一个元素）
  4. list可以对数据进行分页操作，通常第一页的信息来自于list，后面的数据来自于数据库

# 四、set

  • 存储需求：存储大量的数据，在查询方面提供更高的效率
  • 存储结构：能够保存大量的数据，高效的内部存储机制，便于查询
  • set可以用于随机推荐

# 4.1 基本操作

  1. 添加、修改数据

    sadd key v1 v2

  2. 查询全部数据

    smembers key

  3. 删除数据

    srem key v1 v2

  4. 获取集合数据总量

    scard key

  5. 判断集合中是否包含指定数据

    sismember key v

  6. 随机获取集合中指定数量的数据

    srandmember key [count]

  7. 随机获取集合中的某个数据并将该数据移出集合

    spop key

  8. 求两个集合的交、并、差集

    sinter key1 key2 
sunion key1 key2
sdiff key1 key2

  9. 求两个集合的交、并、差集并存储到指定集合中

    sinterstore destination key1 key2
sunionstore destination key1 key2
sidffstore destination key1 key2

  10. 将指定数据从原始集合中移动到目标集合中

    smove source destination member

# 4.2 注意事项

  1. set类型不允许数据重复，如果数据已经存在，后面的操作无效
  2. set虽然与hash的存储结构相同，但无法使用hash中的存储值空间

# 五、sorted_set

  • 存储需求：数据排序有利于数据的有效展示，需要提供一种可以根据自身特征进行排序的方式
  • 存储结构：保证数据可以排序，在set的存储结构基础上添加可排序字段score

# 5.1 基本操作

  1. 添加数据

    zadd key score1 member1 score2 member2

  2. 获取全部数据

    zrange key start stop [withscores]
zrevrange key start stop [withscores]

  3. 删除数据

    zrem key member1 member2

  4. 按条件获取数据

    zrangebyscore key min max [withscores] [limit offset count]
zrevrangebyscore key min max [withscores]

  5. 按条件删除数据

    zremrangebyrank key start stop
zremrangebyscope key min max

  6. 获取集合中数据总量

    zcard key
zcount key min max

  7. 集合交、并操作

    zinterstore destination numkeys k1 k2
zunionstore destination numkeys k1 k2

  8. 获取数据对应的索引（排名）

    zrank key member
zrevrank key member

  9. score值获取与修改

    zscore key member
aincrby key increment member

# 5.2 注意事项

  1. score保存的数据存储空间是64位
  2. score保存的数据也可以是一个双精度的double值，可能会丢精度
  3. scorted_set底层存储是基于set结构的，因此数据不能重复。修改的时候，会覆盖原来的值

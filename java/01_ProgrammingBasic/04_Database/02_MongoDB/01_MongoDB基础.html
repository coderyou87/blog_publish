# 一、 MongoDB的业务应用场景

  • High Performance: 对数据库高并发读写的需求
  • High Storage：对海量数据的高效存储和访问的需求
  • High Scalability, High Availability：对数据库的高扩展性和高可用性的需求

具体的应用：

  1. 社交：使用MongoDB存储用户信息，以及用户发表的社交信息，通过地理位置索引实现附近的人、地点等功能
  2. 游戏：使用MongoDB存储游戏用户、装备、积分等信息，以文档的形式存储，方便查询、存储和访问
  3. 物流：使用MongoDB存储订单信息，由于订单状态在运输过程中会不断更新，以MongoDB内嵌数组的形式来存储，一次查询就能将订单所有的数据全部读取到
  4. 物联网：使用MongoDB存储所有接入的智能设备信息，以及设备汇报的日志信息，并对这些信息进行多维度的分析
  5. 视频：使用MongoDB存储用户信息、点赞互动等数据

以上场景中数据的共同特点：

  1. 数据量巨大
  2. 读写操作频繁
  3. 数据的价值不高，事务要求不高

什么时候选择MongoDB：

  1. 事务要求不高，不涉及复杂关联查询
  2. 需求不确定，数据模型不确定
  3. 大量的文本查询、地理信息查询

# 二、MongoDB简介

​ MongoDB是一个开源、高性能、无模式的文档型数据库，设计的初衷就是简化开发和方便扩展，是一种NoSQL数据库，是与关系型数据库最接近的非关系型数据库。

​ MongoDB支持的数据结构非常松散，存储的数据结构是Bson，类似于Json。因此MongoDB可以存储比较复杂的数据，且比较灵活。

​ MongoDB中的一条记录就是一个文档，它是由字段与值组成的数据结构，MongoDB文档类似于Json，一个文档就是一个对象，字段的数据类型是字符型，值的类型除了基本类型外，还可以是其他文档、普通数组和文档数组。

# 四、数据模型

MongoDB的最小存储单位是文档对象（Document），文档对象相当于MySQL中的行，数据在MongoDB中以BSON格式存储。

BSON（Binary Search Document Format）是一种类似json的二进制存储格式，bson支持文档对象以及数组对象嵌套，同时也支持Date和BinDate类型。BSON具有轻量型、可遍历性以及高效性等特点，可以有效描述结构化与非结构化数据。缺点就是空间利用率不高。

BSON支持的数据类型有：String、Integer、Double、Boolean、array、object、null、date、binary data、regular expression和code。

# 五、MongoDB的特点

  1. 高性能

  2. 高可用性

    MongoDB的赋值工具称为副本集（replica set），它可提供自动故障转移和数据冗余

  3. 高扩展性

    MongoDB提供了水平扩展性作为其核心功能的一部分。分片将数据分布在一组集群的机器上。

  4. 丰富的查询支持

    MongoDB支持服务的查询语言，支持基本的读写操作，也支持数据聚合、文本搜索、地理空间查询等。

  5. 无模式、文档模型

# MySQL日志

# 一、错误日志

  1. 介绍

    记录mysql启动、停止以及运行过程中出现的错误信息，当mysql运行不正常时，应首先查看该日志

    错误日志的三个级别：info、warning、error

  2. 在配置文件中指定错误日志的位置

    修改/etc/my.cnf文件

    log-error=/opt/module/mysql/log/mysql-error.log

  3. 通过SQL查看日志文件的位置

    show variables like 'log_error';

# 二、二进制日志

  1. 介绍

    二进制日志会记录除了查询之外的其他所有操作。

  2. 在配置文件中指定日志位置

    log-bin=/opt/module/mysql/log/mysql-bin.log

  3. 设置二进制日志过期时间

    由于二进制日志比较大，会占用较多空间，所以可以设置日志的过期时间，过期自动删除

    expire_logs_days=5

    或者

    binlog_expire_logs_seconds=5

  4. 查看是否开启了二进制日志

    show variables like 'log_bin';

  5. 查看二进制文件名

    show binary logs;

  6. 回放操作

    show binlog events;

  7. 查看二进制日志

    mysqlbinlog mysql-bin.000001 | more

  8. 将二进制日志导出到文本中

    mysqlbinlog mysql-bin.000001 > mysql-bin.txt
mysqlbinlog --start-position=4 --stop-position=661 mysql-bin.000001 > mysql-bin.txt

  9. 从二进制日志中恢复数据

    1. 使用操作的起始位置和结束位置来恢复数据

      先使用show binlog events;查看操作的起始位置和结束位置。

      mysqlbinlog --start-position=315 --stop-position=7242 mysql-bin.000001 | mysql -uroot -pabcd--

    2. 使用source、起始位置、结束位置来恢复数据

      先转存日志为文本

      mysqlbinlog --start-position=315 --stop-position=7242 mysql-bin.000001 > mysql-bin.txt

      使用source恢复数据

      source /opt/myql-bin.txt

    3. 使用时间段来恢复数据（恢复某一时间段内的数据）

      mysqlbinlog --start-datetime="2020-12-03 23:00:00" --stop-datetime="2020-12-04 00:00:00" mysql-bin.000001 | mysql -uroot -pabcd--

  10. 维护二进制日志

    每次重启mysql服务，会重新生成一个二进制文件；当二进制文件达到限制，会自动生成一个新的二进制文件

    1. 删除所有二进制文件

      reset master

    2. 删除小于某个序号的文件

      删除需要小于17的文件

       purge master logs to ‘mysql-bin.000017′;

    3. 根据配置的过期时间，删除文件

      mysqladmin flush-log

# 三、慢查询日志

  1. 介绍

    记录所有的慢查询日志

  2. 开启查询日志记录

    修改/etc/my.cnf文件

    # 开启慢日志
slow_query_log=on
# 慢日志文件位置
slow-query-log-file=/opt/module/mysql/log/slow-query.log
# 超过1秒为慢查询
long_query_time=1

# 四、通用查询日志

  1. 介绍

    记录用户对数据库的所有操作

  2. 配置

    general_log=on
general_log_file=/opt/module/mysql/log/general_log.log

  3. 删除所有的通用查询日志

    mysqladmin -uroot -pAbcd flush-logs

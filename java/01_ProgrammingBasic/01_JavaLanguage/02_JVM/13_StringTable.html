# StringTable

# 一、String的基本特性

  • String：字符串，使用一对“”引起来表示

    String str1 = "abc";
String str2 = new String("def");

  • String声明为final的，不可被继承

    public final class String
    implements java.io.Serializable, Comparable<String>, CharSequence

  • String实现了Serializable接口，表示字符串是支持序列化的；实现了Comparable接口，表示String可以比较大小

  • String在jdk8及以前，内部定义为 final char[] ；在jdk9修改成 final byte[]

    jdk 1.8

    private final char value[];

    jkd 1.9

    private final byte value[];

  • String在JDK9中存储结构变更

    由于存储在堆中的String对象，大多是拉丁字符，一个拉丁字符只占用一个字节，因此有一半的存储空间被浪费。在JDK9中将char数据该为byte数组，就能节省一半的空间。在存储数据的同时，添加标识，对于编码为拉丁或者ISO-8859-1类型的字符，就使用byte数组存储，对于像中文这样的字符，则使用两个byte来存储一个字符。

  • **结论：**从jdk1.9开始，String使用byte数组来存储数据；与String相关的StringBuilder和StringBuffer也做了同样的修改

  • String的不可变性

    • String代表不可变的字符序列，简称：不可变性
    • 当堆字符串重新赋值时，需要重写指定内存区域赋值，不能使用原有的value进行赋值
    • 当对现有的字符串进行连接操作时，也需要重新指定内存区域赋值
    • 当调用String的replace()方法修改指定字符或字符串时，也需要重新指定内存区域赋值

  • 通过字面量的方式（String str = “abc”）给一个字符串赋值，此时的字符串值声明在字符串常量池中

  • 面试题：

    public class StringTest {

    String str = "good";
    char[] chars = {'t', 'e', 's', 't'};

    public void change(String str, char[] chars) {
        str = "hello string";
        chars[0] = 'b';
    }

    public static void main(String[] args) {
        StringTest stringTest = new StringTest();

        System.out.println(stringTest.str); // good
        System.out.println(stringTest.chars); // test

        stringTest.change(stringTest.str, stringTest.chars);

        System.out.println(stringTest.str);  // good
        System.out.println(stringTest.chars); // best
    }
}

  • 字符串常量池中不会存储相同内容的字符串

  • String的String Pool是一个固定大小的HashTable，其默认长度为1009,。如果在String Pool中存放了过多String，会造成大量的Hash冲突，从而导致链表过长，调用String.intern时，性能会降低

  • 使用**-XX:StringTableSize**可以设置StringTable的长度

  • 在JDK6中，StringTable固定的（长度为1009），如果常量池中字符串过多，就会导致性能下降

  • 在JDK7中，StringTable的长度默认是60013

  • 从JDK8开始，1009是StringTableSize可设置的最小值

# 二、String的内存分配

  • 在Java语言中有8种基本数据类型和String类型。为了让这些类型在程序运行过程中，加载的速度更快以及更加节省内存，都提供了一种常量池的概念。
  • 常量池类似于Java系统级别的缓存。8种基本数据类型的常量池都是系统协调的，String类型的常量池比较特殊，主要的使用方法有两种：
    • 直接使用双引号声明出来的String对象会直接存储在常量池中（String str = “abc”）
    • 使用String类中的**intern()**方法
  • 在jdk6及以前，字符串常量池在永久代（永久代是方法区的具体实现）中
  • 在jdk7中，将字符串常量池由永久代移入堆中（所有的对象都创建在堆上）
  • 在jdk8及以后，永久代由元空间替代，但字符串常量池任然在堆中
  • 为什么要调整字符串常量池的位置？
    • 永久代的空间默认比较小
    • 永久代垃圾回收频率比较低

# 四、字符串拼接操作

  1. 常量与常量的拼接结果存储在常量池（编译期优化）

  2. 常量池中不会存储相同内容的常量

  3. 如果常量和变量进行拼接，则拼接的结果存储在堆中（堆中，没常量池的其他地区）（拼接使用的是StringBuilder）

  4. 如果拼接的结果调用了intern()方法，则主动将常量池中还没有的字符串入池，并返回对象的地址

  5. 面试题1

    private static void test1() {
    String str1 = "a" + "b" + "c";
    String str2 = "abc";
    System.out.println(str1 == str2); //true

    System.out.println(str1.equals(str2)); // true
}

    在Java编译期间，已经将"a" + "b" + "c"进行了拼接，得到了结果“abc”，并存储在常量池中

    String str2 = "abc";，str2直接使用的是常量池的中的“abc”。所以str1与str2的地址比较和值比较都相等。

  6. 面试题2

    private static void test2() {
    String s1 = "javaEE";
    String s2 = "hadoop";
    String s3 = "javaEEhadoop";
    String s4 = "javaEE" + "hadoop";
    String s5 = s1 + "hadoop";
    String s6 = "javaEE" + s2;
    String s7 = s1 + s2;


    // s3先创建并存储在常量池中
    // s4由于是两个常量做拼接，在java编译期，就会将两个常量拼接完成并将结果存储在常量池中
    // 由于s3与s4的值相同，所有存储s4的时候，会存储失败，s4会指向s3所指向的内存地址
    System.out.println(s3 == s4); // true

    // s5、s6、s7在拼接过程中，都有变量的参与，所以拼接的结果会存储在堆中非常量池的地方
    System.out.println(s3 == s5); // false
    System.out.println(s3 == s6); // false
    System.out.println(s3 == s7); // false
    System.out.println(s5 == s6); // false
    System.out.println(s5 == s7); // false
    System.out.println(s6 == s7); // false

    // 当s6调用intern方法时，会将字符串 “javaEEhadoop”存储到常量池中
    // 由于常量池中已经存在了字符串 “javaEEhadoop”，所以存储失败，
    // 并返回常量池中字符串 ”javaEEhadoop“的地址，所以s3与s8指向的是同一个内存区域
    String s8 = s6.intern();
    System.out.println(s3 == s8); // true
}

    // 被final修饰，s1已经是常量了
    final String s1 = "a";
    // 被final修饰，s2也是常量
    final String s2 = "b";

  8. 字符串拼接细节

    private static void test4() {
    String s1 = "a";
    String s2 = "b";
    /**
     * 1、 StringBuilder builder = new StringBuilder();
     * 2、 builder.append(s1);
     * 3、 builder.append(s2);
     * 4、 builder.toString();
     */
    String s3 = s1 + s2;
}

  9. 如果拼接符号两边都是常量或者常量引用，则任然使用编译器优化，不会使用StringBuilder

  10. 字符串拼接比较分析

    private static void test6() {
    String str = "";
    for (int i = 0; i < 10000; i++) {
        str += "a";
    }
}

    private static void test7() {
    StringBuilder builder = new StringBuilder();
    for (int i = 0; i < 10000; i++) {
        builder.append("a");
    }
    builder.toString();
}

    test6比test7慢很多

    分析：

    • 使用String进行拼接，会创建大量的StringBuilder和String对象，占用内存更多，如果发生GC，则占用更多时间
    • 使用StringBuilder进行拼接，只会创建一个StringBuilder对象

    在创建StringBuilder对象时，可以指定初始化数组长度。

# 五、intern()的使用

定义：

public native String intern();

当程序中使用字符串对象调用intern方法时，intern方法会判断字符串常量池中是否存在当前字符串，如果存在，则直接返回字符串常量池中关于该字符串的引用；如果不存在，则将当前字符串添加到常量池中并返回常量池中关于该字符串的引用。

如何保证引用指向的是常量池中的数据：

  1. 使用字面量

    String str = "hello";

  2. 使用intern方法

    String str = new String("hello").intern();

面试题：

private static void test8() {
    // s指向堆中的数据（非常量池）
    String s = new String("1");
    s.intern();
    // s2指向常量池中的数据
    String s2 = "1";
    System.out.println(s == s2);

    // s3 指向的是常量池中的数据
    // new String("1") + new String("1"); 该语句最后执行的步骤是调用StringBuilder中的toString方法
    // 由于toString方法并不会将字符串存储到常量池中，所以此时常量池中不会有字符串"11"
    String s3 = new String("1") + new String("1");
    // 调用intern方法时，会在常量池中创建一个引用，该引用会指向堆中的字符串"11"（这样做是为了节省空间，避免在堆中和常量池中存储相同的数据）
    s3.intern();
    // s4指向的也是常量池中的数据
    // 由于常量池中已经存在字符串"11"的引用，所以为s4赋值的时候，会返回字符串"11"的引用（字符串"11"实际存储于堆中）
    String s4 = "11";
    // s3和s4都指向于堆中的字符串"11"，所以s3与s4的地址比较是相同的
    System.out.println(s3 == s4);
}

使用new String(“ab”)创建String对象时，会将字符串分别存储在堆中和字符串常量池中，二者之间没有联系。

private static void test10() {
    // 使用此构造函数创建String对象，会将字符串分别存储到堆中和常量池中
    // 此时str1指向的是堆中的字符串
    String str1 = new String("11");
    // 调用intern方法会将字符串存储到常量池中，并返回字符串在常量池中的引用
    // 如果字符串已经存在于常量池中，则直接返回该字符串在常量池中的引用
    // 此时返回的字符串在常量池中的引用，没有被使用，str1任然指向堆中的字符串
    str1.intern();
    // 使用字面量创建字符串对象，因为常量池中已经存在，所以使用的是常量池中已经存在的字符串
    String str2 = "11";
    // str1与str2所指向的地址不同
    // str1指向的是堆中的字符串
    // str2指向的是常量池中的字符串
    System.out.println(str1 == str2);
}

使用public String(char value[], int offset, int count)创建String对象时，只会将字符串存储在堆中，不会将字符串存储到常量池中（StringBuilder类中的toString，就是采用的这种方式）。当调用intern方法时，只会在常量池中创建一个引用，该引用指向堆中的字符串。如果后面使用了字面量创建相同的字符串，则也会指向堆中的字符串。

private static void test9() {
    char[] chars = {'1', '1'};
    // 创建一个String对象，字符串只会存储在堆中，不会存储到字符串常量池中
    String str1 = new String(chars, 0, 2);
    // 在常量池中创建一个引用，指向堆中的字符串
    str1.intern();
    // 引用常量池中存在字符串"11"的引用，所以不会重新创建字符串
    // str2也会指向堆中的字符串，与str1的指向相同
    String str2 = "11";
    // 所以str1与str2指向的内存地址相同
    System.out.println(str1 == str2);
}

面试题：

private static void test11() {
    // 创建一个String对象，字符串只会存储在堆中，不会存储到常量池中
    String str1 = new String("1") + new String("2");
    // 创建一个String对象，字符串存储在常量池中
    String str2 = "11";
    // 将str1所指向的字符串存储到常量池中，由于常量池中已经存在"11"，所以intern方法会返回str2所指向的地址
    //  但是返回的地址并没有被使用,所以str1任然指向堆中的字符串
    str1.intern();
    // 所以str1与str2所指向的内存地址不一样
    System.out.println(str1 == str2);
}

总结：

  1. JDK1.6中，intern方法是尝试将字符串放入常量池（此时的常量池在永久代）：
    • 如果常量池中没有，会拷贝一份字符串放入常量池中，并返回常量池中该字符串的地址
    • 如果常量池中已经存在该字符串，则不会再放入该字符串，而是直接返回常量池中关于该字符串的地址
  2. JDK1.7以后，intern方法的作用是尝试将字符串放入常量池（此时的常量池在堆内）：
    • 如果常量池中没有，会把字符串在堆中的引用地址拷贝一份，放入常量池中，并返回常量池中的该引用的地址
    • 如果常量池中已经存在该字符串，则不会再放入该字符串，而是直接返回常量池中关于该字符串的地址

练习1：

private static void test12() {
    // str1指向的是堆中的字符串，此时常量池中还没有"ab"
    String str1 = new String("a") + new String("b");
    // 拷贝堆中字符串"ab"的引用，将引用放入常量池中，并返回常量池中关于该引用的地址
    String str2 = str1.intern();
    // 通过字面量声明一个String对象，发现常量池中已经存在字符串"ab"的引用，则直接返回该字符串的引用
    // 所以str1与字符串“ab”所指向的地址相同（使用的都是堆上的字符串）
    System.out.println(str1 == "ab");
    // 通过字面量声明一个String对象，发现常量池中已经存在字符串"ab"的引用，则直接返回该字符串的引用
    // str2与字符串"ab"都是使用的堆上的字符串
    System.out.println(str2 == "ab");
}

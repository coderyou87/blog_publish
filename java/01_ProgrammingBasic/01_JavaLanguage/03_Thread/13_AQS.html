# AbstractQueuedSynchronizer

# 一、LockSupport

用于创建锁和其他同步类的基本线程阻塞原语。

类中的两个主要方法park与unpark，类似于wait与notify。

LockSupport是一个线程阻塞工具类，所有的方法都是静态方法，可以让线程在任意位置阻塞，也可以唤醒线程，其底层是调用的Unsafe中的native方法。

LockSupport和每一个使用它的线程都有一个许可证（permit），permit的值只能是1或0，默认是0，当调用unpark方法时，permit会变为1，unpark重复调用无效。调用park方法时，permit会变为0。

当调用park时，如果有凭证（permit为1），则消费凭证（permit=1）；如果没有凭证，则阻塞等待

当调用unpark时，如果没有凭证（permit=0），则生产凭证（permit=1）；如果有凭证（permit=1），则不生产

如果需要park多次，需要多个线程来 unpark，不然会出现阻塞，因为一个线程只能执行一次 unpark。

面试题：

  1. 为什么可以先唤醒线程后阻塞线程

    因为unpark后，会生成一个凭证，当调用park方法时，就能够消费凭证，而不会阻塞

  2. 为什么唤醒两次后，再阻塞两次，最终结果还是阻塞

    因为凭证的数量最多为1（permit=1），连续两次调用unpark，permit的值还是1，当第一次调用park时，就消费了凭证（permit=0），第二次调用park时，由于没有凭证，所以线程会进入阻塞状态。

阻塞与唤醒线程的方法：

  1. 使用Object类中的wait方法阻塞线程，notify方法唤醒线程
  2. 使用Condition类中的await方法阻塞线程，signal方法唤醒线程
  3. 使用LockSupport类中的park方法阻塞线程，unpark方法唤醒线程

# 1.1 使用wait与notify实现线程的阻塞与唤醒

wait与notify必须成对出现，都只能出现在Synchronized同步代码中，wait必须比notify先出现。

private static void waitAndNotify() {
    new Thread(() -> {
        try {
            TimeUnit.SECONDS.sleep(3);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

            System.out.println(Thread.currentThread().getName() + "\t" + "come in");
            try {
                objectLock.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(Thread.currentThread().getName() + "\t" + "notify");
        }
    }, "A").start();

    new Thread(() -> {
        synchronized (objectLock) {
            System.out.println(Thread.currentThread().getName() + "\t" + "notice");
            objectLock.notify();
        }
    }, "B").start();
}

wait与notify需要在同步代码块中使用，并且需要成对出现

# 二、AQS内部结构

抽象队列同步器（AbstractQueuedSynchronizer）：是用来构建锁或者其他同步器组件的重量级基础框架及整个JUC体系的基石，通过内置的FIFO队列来完成资源获取线程的排队工作，并通过一个int类型变量表示持有锁的状态

CLH队列：使用单向链表来模拟的双向队列（FIFO）

和AQS相关的类：

锁与同步器：

  1. 锁：面向锁的使用者，定义了使用者与锁交互的API
  2. 同步器：面向锁的实现者，在java规范中，简化了锁的实现，屏蔽了同步状态管理，阻塞线程排队和通知、唤醒机制等

在多线程加锁的情况下，加锁会导致阻塞，对于阻塞的线程，需要对其进行排队，以方便管理。

在多线程情况下，抢到资源的线程直接使用资源来处理业务，对于其他没有抢到资源的线程，需要排队等候，它们仍然保留抢占资源的可能。

在排队等候的时候，形成了队列。在AQS中，采用了一种名为CLH的队列（FIFO）。它将等待的线程封装成队列的节点（Node），通过CAS、自旋以及LockSupport.park()的方式，维护资源的状态，使并发达到同步的控制效果。

AQS中的同步状态变量：

/**
 * The synchronization state.
 */
private volatile int state;

如果state等于0，说明当前资源没有被占用；如果等于或大于1，说明当前资源已经被线程占用，后面的线程需要到CLH队列中等待。

AQS中的Node：

static final class Node {
    // 共享
    static final Node SHARED = new Node();
    // 独占
    static final Node EXCLUSIVE = null;
    // 线程被取消
    static final int CANCELLED = 1;
    // 后继线程需要唤醒
    static final int SIGNAL = -1;
    // 等待condition唤醒
    static final int CONDITION = -2;
    // 共享式同步状态获取将会无条件地传播下去
    static final int PROPAGATE = -3;
    // 初始为0，状态是上面几种
    volatile int waitStatus;
    // 前置节点
    volatile Node prev;
    // 后置节点
    volatile Node next;
    // 当前线程
    volatile Thread thread;
    //
    Node nextWaiter;

    final boolean isShared() {
        return nextWaiter == SHARED;
    }

    final Node predecessor() throws NullPointerException {
        Node p = prev;
        if (p == null)
            throw new NullPointerException();
        else
            return p;
    }

    Node() {    // Used to establish initial head or SHARED marker
    }

    Node(Thread thread, Node mode) {     // Used by addWaiter
        this.nextWaiter = mode;
        this.thread = thread;
    }

    Node(Thread thread, int waitStatus) { // Used by Condition
        this.waitStatus = waitStatus;
        this.thread = thread;
    }
}

    final Thread current = Thread.currentThread();

    final Thread current = Thread.currentThread();
    int state = getState();
    // 如果c等于0， 说明当前资源没有被占用，可以直接使用
    if (state == 0) {
        //hasQueuedPredecessors用于判断等待队列中是否存在有效节点
        if (!hasQueuedPredecessors() && compareAndSetState(0, acquires)) {
            setExclusiveOwnerThread(current);
            return true;

        if (nextState < 0) {
            throw new Error("Maximum lock count exceeded");
        }
        setState(nextState);
        return true;
    }
    return false;
}

    public final void acquire(int arg) {
    // tryAcquire：尝试获取锁，arg=1
    // tryAcquire() 返回true，表示获取锁成功，返回false，表示获取锁失败
    // 如果获取锁成功，结束当前程序
    // 如果获取锁失败，则进入队列等候
    if (!tryAcquire(arg) && acquireQueued(addWaiter(Node.EXCLUSIVE), arg)) {
        selfInterrupt();
    }
}

    调用Sync类中的nonfairTryAcquire(int acquires)：

    final boolean nonfairTryAcquire(int acquires) {
    // 获取当前线程
    final Thread current = Thread.currentThread();
    // 获取当前资源状态，c等于0，表示资源没有被占用；c大于1，表示当前资源已经被占用
    int state = getState();
    if (state == 0) {
        // 当前资源没有被占用（第二个线程进入的时候，第一个线程刚好办理完业务，第二个线程立即利用资源办理业务）
        if (compareAndSetState(0, acquires)) {
            // 将当前线程设置为占用资源的线程
            setExclusiveOwnerThread(current);
            // 返回true表示获取锁成功
            return true;
        }
    } else if (current == getExclusiveOwnerThread()) {
        // current == getExclusiveOwnerThread() 表明某一个线程刚办理完业务，并释放资源
        // 此时此线程再次参与竞争资源，并且竞争成功
        // nextState = state + acquires 是可重入锁的体现
        int nextState = state + acquires;
        // overflow
        if (nextState < 0) {
            throw new Error("Maximum lock count exceeded");
        }
        setState(nextState);
        // 返回true，表示获取锁成功
        return true;
    }
    // 返回false，表示获取锁失败
    return false;
}

  6. AbstractQueuedSynchronizer类中的挂载节点方法（入队）addWaiter(Node mode)

    // 将当前线程加入到等待队列中
private Node addWaiter(Node mode) {
    // 创建一个node节点
    Node node = new Node(Thread.currentThread(), mode);
    // Try the fast path of enq; backup to full enq on failure
    Node pred = tail;
    // 如果尾结点不为空，将node入队，并将尾指针指向node
    if (pred != null) {
        node.prev = pred;
        if (compareAndSetTail(pred, node)) {
            pred.next = node;
            return node;
        }
    }
    // 节点入队
    enq(node);
    return node;
}

    入队 enq(final Node node) 方法：

    private Node enq(final Node node) {
    // 自旋
    for (; ; ) {
        Node t = tail;
        // 如果尾节点为空，则使用cas初始化头节点
        if (t == null) { // Must initialize
            // 初始化头结点后，头结点的参数waitStatus为0，thread为null
            // 头结点被称为哨兵节点，只是用于占位
            if (compareAndSetHead(new Node()))
                // 将尾指针执行头节点
                tail = head;
        } else {
            // 将node节点入队
            node.prev = t;
            // 将尾指针指向node节点
            if (compareAndSetTail(t, node)) {
                t.next = node;
                return t;
            }
        }
    }
}

  7. AbstractQueuedSynchronizer类中的acquireQueued(final Node node, int arg)方法

    final boolean acquireQueued(final Node node, int arg) {
    // failed如果是true，则取消排队
    boolean failed = true;
    try {
        // 标明程序是否被中断
        boolean interrupted = false;
        // 自旋
        for (; ; ) {
            // 获取node的前置节点
            final Node prev = node.predecessor();
            // 如果prev等于前置节点
            // tryAcquire 入队后，再次尝试获取锁
            if (prev == head && tryAcquire(arg)) {
                // 线程被唤醒后，并且获取到锁
                // 将node设置为哨兵节点
                setHead(node);
                // prev.next是原来的哨兵节点，此处原来的哨兵节点出队
                // help GC
                prev.next = null;
                // 将failed设置为false，表明当前线程没有被取消排队
                failed = false;
                // interrupted为false，表明当前线程没有被中断
                return interrupted;
            }
            // 获取锁失败
            // 如果prev是哨兵节点，node是第一个进入队列的节点，shouldParkAfterFailedAcquire会将prev的wait-Status设置为-1，并返回false
            // 第二次循环，shouldParkAfterFailedAcquire返回true
            // 正常情况下，parkAndCheckInterrupt 会阻塞当前线程，在当前线程被unpark后，返回false
            // 进入下一次自旋
            if (shouldParkAfterFailedAcquire(prev, node) && parkAndCheckInterrupt()) {
                interrupted = true;
            }
        }
    } finally {
        // failed 是true，取消排队
        if (failed) {
            cancelAcquire(node);
        }
    }
}

  8. 将当前线程设置为阻塞状态

    shouldParkAfterFailedAcquire(Node pred, Node node)

    // 获取锁失败后，需要将当前线程线程的状态设置为阻塞
private static boolean shouldParkAfterFailedAcquire(Node pred, Node node) {
    // 前置节点的等待状态
    int ws = pred.waitStatus;
    // 通过自旋，哨兵节点的waitStatus等于Node.SIGNAL(-1)，直接返回true
    if (ws == Node.SIGNAL) {
        /*
         * This node has already set status asking a release
         * to signal it, so it can safely park.
         */
        return true;
    }

    if (ws > 0) {
        /*
         * Predecessor was cancelled. Skip over predecessors and
         * indicate retry.
         */
        do {
            node.prev = pred = pred.prev;
        } while (pred.waitStatus > 0);
        pred.next = node;
    } else {
        /*
         * waitStatus must be 0 or PROPAGATE.  Indicate that we
         * need a signal, but don't park yet.  Caller will need to
         * retry to make sure it cannot acquire before parking.
         */
        // 将哨兵节点（头结点）的waitStatus设置为-1
        compareAndSetWaitStatus(pred, ws, Node.SIGNAL);
    }
    return false;
}

    parkAndCheckInterrupt()

    private final boolean parkAndCheckInterrupt() {
    // 设置当前线程为阻塞状态，至此，当前线程不再尝试获取锁，直到被唤醒
    // 但是程序并没有结束，一旦被unpark，则会继续执行程序
    LockSupport.park(this);
    // 如果线程没有被中断，返回false
    return Thread.interrupted();
}

public final boolean release(int arg) {
    // 释放资源，设置waitStatus的值为0，设置资源的占有线程为null
    if (tryRelease(arg)) {

  5. tryRelease方法在Sync类中的具体实现

    protected final boolean tryRelease(int releases) {
    // releases 等于1
    // 正常情况下，做减法后，state等于0
    int state = getState() - releases;

    if (Thread.currentThread() != getExclusiveOwnerThread()) {
        throw new IllegalMonitorStateException();
    }
    // free标识当前资源是否被释放
    boolean free = false;
    if (state == 0) {
        // 设置当前资源的状态为可用状态
        free = true;
        // 当前线程的业务办理完毕
        // 设置当前资源的占用线程为null，即没有被线程占用
        setExclusiveOwnerThread(null);
    }
    setState(state);
    return free;
}

  6. 唤醒等候线程的unparkSuccessor(Node node)方法

    private void unparkSuccessor(Node node) {

    // ws等于-1
    int ws = node.waitStatus;
    if (ws < 0) {
        // 将ws设置为0（创建节点时的初始状态）
        compareAndSetWaitStatus(node, ws, 0);
    }
    
    // 第一个等候节点
    Node successor = node.next;
    // successor.waitStatus = 0
    if (successor == null || successor.waitStatus > 0) {
        successor = null;
        for (Node tailNode = tail; tailNode != null && tailNode != node; tailNode = tailNode.prev) {
            if (tailNode.waitStatus <= 0) {
                successor = tailNode;
            }
        }
    }

    if (successor != null) {
        // 唤醒第一个等候节点
        LockSupport.unpark(successor.thread);
    }
}

# Jstat

官网文档地址

作用: 监视Java虚拟机各种运行状态信息, 可以显示本地或远程虚拟机进程中的类装载, 内存, 垃圾收集, JIT编译等数据

命令格式: jstat [options vmid [interval] [count] ]

参数解释:

options: 用户希望查看的虚拟机信息, 主要分为: 类状态, 垃圾收集, 运行期编译情况

  1. -class: 显示有关类加载行为的统计信息
  2. -compiler: 显示有关JVM即时编译的统计信息
  3. -gc: 显示有关垃圾回收的统计信息
  4. -gccapacity: 显示堆内存的空间信息
  5. -gccause: 显示有关垃圾收集的统计信息(同 -gcutil), 以及上一次和当前垃圾收集事件的原因
  6. -gcnew: 显示新生代的垃圾回收信息
  7. -gcnewcapacity: 显示新生代内存统计信息
  8. -gcold: 显示老年代垃圾回收信息
  9. -gcoldcapacity: 显示老年代内存统计信息
  10. -gcmetacapacity: 显示元空间的统计信息
  11. -gcutil: 显示有关总的垃圾回收统计信息
  12. -printcompilation: 显示JVM方法编译的统计信息

vmid:

  1. 如果查看本地虚拟机进程, vmid等于本地虚拟机唯一ID

  2. 如果是远程虚拟机进程, 则vmid的格式为:

    protocol:lvmid[@hostname] [:port] /servername

interval: 采样间隔, 单位是秒或毫秒, 默认单位是毫秒, 必须是正整数

count: 要显示的样本数

如果不指定interval与count的值, 默认只查询一次

# jstat -class

  • Loaded: 加载的class数量
  • Bytes: class所占用的空间大小
  • Unloaded: 未加载的class数量
  • Bytes: 未加载的class所占用的空间大小
  • Time: 时间

# jstat -compiler

  • Compiled：编译数量。
  • Failed：失败数量
  • Invalid：不可用数量
  • Time：时间
  • FailedType：失败类型
  • FailedMethod：失败的方法

# jstat -gc

  • S0C：第一个幸存区的大小
  • S1C：第二个幸存区的大小
  • S0U：第一个幸存区的使用大小
  • S1U：第二个幸存区的使用大小
  • EC：伊甸园区的大小
  • EU：伊甸园区的使用大小
  • OC：老年代大小
  • OU：老年代使用大小
  • MC：方法区大小
  • MU：方法区使用大小
  • CCSC:压缩类空间大小
  • CCSU:压缩类空间使用大小
  • YGC：年轻代垃圾回收次数
  • YGCT：年轻代垃圾回收消耗时间
  • FGC：老年代垃圾回收次数
  • FGCT：老年代垃圾回收消耗时间
  • GCT：垃圾回收消耗总时间

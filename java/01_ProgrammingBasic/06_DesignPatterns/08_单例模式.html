# 单例模式

# 一、介绍

一个单一的类，负责创建自己的对象，同时确保系统中只有单个对象被创建。

单例特点

  • 某个类只能有一个实例；（构造器私有）
  • 它必须自行创建这个实例；（自己编写实例化逻辑）
  • 它必须自行向整个系统提供这个实例；（对外提供实例化方法）

# 二、应用场景与面试问题

应用场景：

  1. 多线程中的线程池
  2. 数据库连接池
  3. 系统环境信息
  4. 上下文（ServletContext）

面试问题：

  1. 系统环境问题（System.getProperties()）
  2. Spring 中怎么保证组件单例
  3. ServletContext 是什么？是单例吗？怎么保证？
  4. ApplicationContext 是什么？是单例吗？怎么保证？
    1. ApplicationContext——tomcat：一个应用（部署一个 war 包）会有一个应用上下文
    2. ApplicationContext——Spring：表示整个 IOC 容器，怎么保证单例；IOC 容器中有很多组件，怎么保证单例？
  5. 数据库连接池一般怎么创建的，如何保证单实例？

# 三、实现

  • 俄汉式（静态常量）：

    优点：实现比较简单，在类装载的时候就完成了实例化，避免了多线程问题

    缺点：对象在类装载的时候就完成了实例化，没有达到懒加载的效果。如果该对象一直没有被使用，则存在内存浪费。

    public class Singleton {

    private static final Singleton instance = new Singleton();

    private Singleton() {

    }

    public static Singleton getInstance() {
        return instance;
    }
}

  • 饿汉式（静态代码块）：

    优点：实现比较简单，在类装载的时候就完成了实例化，避免了多线程问题

    缺点：对象在类装载的时候就完成了实例化，没有达到懒加载的效果。如果该对象一直没有被使用，则存在内存浪费。

    public class Singleton {

    private static final Singleton instance;

    static {
        instance = new Singleton();
    }

    private Singleton() {

    }

    public static Singleton getInstance() {
        return instance;
    }
}

  • 懒汉式：

    懒汉式能够达到懒加载的效果，但是在多线程情况下，存在线程安全问题

    public class Singleton {

    private static Singleton instance;

    private Singleton() {

    }

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

  • 懒汉式（使用 Synchronized 在方法上加锁）：

    优点：没有线程安全问题

    缺点：锁粒度太大，影响效率

    public class Singleton {

    private static Singleton instance;

    private Singleton() {

    }

    public static synchronized Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

  • 懒汉式（对代码块加锁）

    因为对象的创建在计算机底层被分为三个步骤，所以此方式仍然存在线程安全问题

    public class Singleton {

    private static Singleton instance;

    private Singleton() {

    }

    public static Singleton getInstance() {
        synchronized (Singleton.class) {
            if (instance == null) {
                instance = new Singleton();
            }
            return instance;
        }
    }
}

    public class Singleton {

    private static Singleton instance;

    private Singleton() {

    }

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class){
                instance = new Singleton();
            }
        }
        return instance;
    }
}

  • 双检锁

    此方式由于线程之间的变量不可见性，任然存在线程安全问题

    public class Singleton {

    private static Singleton instance;

    private Singleton() {

    }

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

    使用 Volatile 关键字修饰 instance，解决线程不可见问题。

    优点：线程安全、懒加载、效率高

    public class Singleton {

    private static volatile Singleton instance;

    private Singleton() {

    }

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

  • 静态内部类

    使用静态内部类实现单例与俄汉式有相似之处，也有不同。两者都采用了类状态机制来保证只有一个线程来初始化实例。不同的是，饿汉式在装载 Singleton 时，就会初始化 instance，没有懒加载的功能；而静态内部类方式，在装载 Singleton 时，不会加载其内部类 SingletonInstance，只有在调用 getInstance 方法时，才会初始化 INSTANCE。

    类的静态属性，只有在第一次加载类的时候初始化，所以 JVM 保证了线程的安全性，在 SingletonInstance 初始化的时候，别的类是无法进入的。

    优点：避免了线程安全问题，延迟加载，效率高。

    public class Singleton {

    private Singleton() {

    }

    private static class SingletonInstance {
        // 在Singleton类被装载的时候，SingletonInstance不会被装载
        // 只有调用getInstance方法时，才会装载SingletonInstance
        private static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance() {
        return SingletonInstance.INSTANCE;
    }
}

  • 枚举

    使用枚举来实现单例，不仅能避免线程安全问题，还能防止反序列化重新创建对象。

    public enum Singleton {

    INSTANCE;

}

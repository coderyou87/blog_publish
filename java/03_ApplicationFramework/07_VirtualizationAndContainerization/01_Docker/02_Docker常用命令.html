# Docker 常用命令

Docker命令手册

# 一、帮助命令

  1. 查看 docker 版本

    docker version

  2. 查看 docker 信息

    docker info

  3. 查看 docker 帮助信息

    docker --help

# 二、镜像命令

# 2.1 查看 docker 镜像信息

  • 列出本地主机上的镜像信息

    docker images

  • 列出本地主机上所有的镜像信息(含中间映像层)

    docker images -a

  • 只显示镜像 Id

    docker images -q

  • 显示镜像的摘要信息

    docker images --digests

  • 显示完整的镜像信息

    docker images --no-trunc

  • 查看镜像详情

    docker inspect imageId

# 2.2 搜索镜像信息

官网地址

检索镜像信息:

docker search [options] redis

显示完整的镜像信息

docker search --no-trunc redis

使用 filter 添加查询条件

# stars大于100的redis镜像
docker search --filter stars=100 redis
# 列出自动构建的镜像信息
docker search --filter "is-automated=true" redis
docker search --filter "is-automated=true" --filter "stars=100" redis

# 2.3 拉取镜像

如果不写 Tag,默认为 latest

docker pull redis[:TAG]

# 2.4 删除镜像

  • 删除单个镜像

    docker rmi imageId

  • 强制删除镜像

    docker rmi -f imageId

  • 删除多个镜像

    docker rmi -f imageNmae1:TAG1 imageName2:TAG2

  • 删除全部镜像

    docker rmi -f ${docker images -qa}

# 2.5 根据当前容器创建一个新镜像

docker commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]]

参数：

-a, --author string    Author (e.g., "John Hannibal Smith <hannibal@a-team.com>")
-c, --change list      Apply Dockerfile instruction to the created image
-m, --message string   Commit message
-p, --pause            Pause container during commit (default true)

docker commit -a author_xiaoyou -m "first commit " myNginx my_nginx:v1

如果上面的提交命令执行了两次（提交时的镜像名称相同），之前提交的镜像就会变成游离镜像（不能被使用）

移除游离镜像

docker image prune

# 2.7 将镜像推送到远程仓库

  1. 登陆

    docker login --username=六月徂暑 registry.cn-beijing.aliyuncs.com

    回车，输入密码

    登陆成功后，会将认证信息保存在/root/.docker/config.json文件中

  2. 修改tag

    替换ImageId个ImageVersion

    docker tag ImageId registry.cn-beijing.aliyuncs.com/xiaoyou_namespace/mynginx:ImageVersion

    docker tag 688e73f7e34c registry.cn-beijing.aliyuncs.com/xiaoyou_namespace/mynginx:v2

  3. 推送

    docker push registry.cn-beijing.aliyuncs.com/xiaoyou_namespace/mynginx:ImageVersion

    docker push registry.cn-beijing.aliyuncs.com/xiaoyou_namespace/mynginx:v2

# 2.8 导入导出镜像

  1. 导出

    docker save [OPTIONS] IMAGE [IMAGE...]

    -o, --output string   Write to a file, instead of STDOUT

    docker save -o /opt/software/redis-save.tar redis:latest

  2. 导入

    docker load [OPTIONS]

    -i, --input string   Read from tar archive file, instead of STDIN
-q, --quiet          Suppress the load output

    docker load -i /opt/software/redis-save.tar

# 三、容器命令

容器的状态：

  • Created（创建），使用docker create命令创建容器后的状态
  • Up（运行中），使用docker start或docker run命令运行容器后的状态
  • Pause（暂停），使用docker pause命令暂停容器后的状态
  • Exited（退出），使用docker stop或docker kill命令停止容器后的状态

# 3.1 启动容器

docker run [Options] IMAGE [Command] [Args]
docker run --name=testdocker -d -p 8080（宿主机端口）:7878（容器端口） docker/testdocker

第一个端口是宿主机的端口，必须唯一；第二个是容器的端口，不同容器可以重复

宿主机的多个端口可以绑定到容器的一个端口，但是不允许用宿主机的一个端口去绑定容器中的多个端口

docker run 默认是前台启动

docker run -d 等价于 docker create + docker start

  • --name: 指定容器运行时的名称
  • -d: 后台运行,并返回容器 Id
  • -p: 指定端口映射
  • -P: 随机端口映射
  • -i: 以交互式运行容器
  • -t: 为容器分配一个伪终端

在运行容器的时候，指定网络模式

我们在使用docker run创建Docker容器时，可以用--net选项指定容器的网络模式，Docker有以下4种网络模式：
host模式，使用--net=host指定。
container模式，使用--net=container:NAME_or_ID指定。
none模式，使用--net=none指定。
bridge模式，使用--net=bridge指定，默认设置。
 
默认选择bridge。

# 3.2 创建容器

docker create [OPTIONS] IMAGE [COMMAND] [ARG...]
docker create --name myRedis -p 6379:6379 redis:latest

docker create只能创建容器，不能启动容器

在创建完成后，需要使用以下命令启动容器

docker start containerName
docker start containerId

# 3.3 查看当前运行的容器

docker ps [Options]
  • -a: 列出当前所在运行的容器以及历史上运行过的容器
  • -l: 显示最近创建的容器
  • -n: 显示最近创建的 N 个容器
  • -q: 只显示容器编号
  • --no-trunc: 不截断输出

# 3.4 退出容器

停止容器并退出

exit

不停止容器退出

ctrl + P + Q

# 3.5 启动容器

docker start 容器名字或Id

# 3.6 重启容器

docker restart 容器名字或ID

# 3.7 停止容器

  1. 优雅停机（处理完任务才停机）

    docker stop 容器名字或Id

  2. 强制停止容器

    强制停机（直接拔电源）

    docker kill 容器名字或ID

# 3.8 删除已停止的容器

  • 删除一个容器

    docker rm 容器Id

  • 删除多个容器

    docker rm -f ${docker ps -a -q}
docker ps -a -q xargs docker rm

# 3.9 暂停容器

docker pause 容器名字或ID

# 3.10 恢复容器

docker unpause 容器名字或ID

# 3.11 检查容器里面文件系统的更改

docker diff 容器名字或ID

检查结果：

  • A：添加文件或目录
  • D：删除文件或目录
  • C：更改文件或目录

# 3.12 导入导出容器（export、import）

  1. 导

    docker export [OPTIONS] CONTAINER

    参数

    -o, --output string   Write to a file, instead of STDOUT

    docker export -o /opt/software/nginx-export.tar myNginx

  2. 导入

    docker import [OPTIONS] file|URL|- [REPOSITORY[:TAG]]

    -c, --change list       Apply Dockerfile instruction to the created image
-m, --message string    Set commit message for imported image
	--platform string   Set platform if server is multi-platform capable

    docker import /opt/software/nginx-export.tar my_nginx:v3

    以这种方式导入的镜像，不能直接运行：

    docker run -d -P --name=myNginxV3 my_nginx:v3

    会出现以下错误：

    需要先找到源容器的启动命令：

    docker inspect myNginx

    或者执行以下命令找运行命令：

    docker ps --no-trunc


    "/docker-entrypoint.sh nginx -g 'daemon off;'"

    导入容器的最终启动命令为：

    docker run -d -P my_nginx:v3 /docker-entrypoint.sh nginx -g 'daemon off;'

# 四、其他重要命令

# 4.1 启动守护式容器

docker run -d imageName

# 4.2 查看容器日志

docker logs -f -t -tail 容器Id
  • -t：加入时间戳
  • -f：跟随最新的日志答打印
  • -tail n：显示最后 n 行

# 4.3 查看容器内运行的进程

docker top 容器Id

# 4.4 查看容器内运行的细节

docker inspect 容器Id

# 4.5 进入正在运行的容器并以命令行交互

docker exec -it 容器Id bashShell
docker attach 容器Id
# 以特权（root）进入容器
docker exec -it -u 0:0 --privileged myNginx /bin/bash

exec：在容器中打开新的终端，并且可以启动新的进程

attach：直接进入容器启动命令的终端，不会启动新的进程

# 4.6 在容器与主机之间拷贝文件

docker cp [OPTIONS] CONTAINER:SRC_PATH DEST_PATH|-
docker cp [OPTIONS] SRC_PATH|- CONTAINER:DEST_PATH

  1. 将容器中的文件，拷贝到宿主机

    docker cp 容器Id：容器内路径 目的主机路径
docker cp 9ecd69dead67:/etc/yum.repos.d/docker_repos   ./

  2. 将宿主机的文件，拷贝到容器中

    docker cp  目的主机路径  容器Id：容器内路径
docker cp ./nginx.tar.gz   9ecd69dead67:/opt

# MyBatis参数处理

# 一、单个参数

在xml文件中，使用#{参数名}来获取传入的参数值

  1. 接口：

    List<PmsAttr> queryAttr(@Param("id") String id);

  2. xml文件：

    在指定映射类型时，使用类的全限定名称

    <select id="queryAttr" resultType="com.it.mybatis.bean.PmsAttr">
    select *
    from pms_attr
    where attr_id = #{id}
</select>

    使用别名

    在application.properties中添加配置：

    mybatis.type-aliases-package=com.it.mybatis.bean

    <select id="queryAttr" resultType="pmsAttr">
    select *
    from pms_attr
    where attr_id = #{attrId}
    and attr_name = #{attrName}
</select>

# 二、多个参数

# 2.1 多个分散的参数

在多个参数的情况下，MyBatis会讲参数封装在一个Map中。map的key是param1、param2..。如果参数过多，使用param有时不便于区分参数。因此，我们可以在接口上使用@Param注解，明确指定参数名称，也就是map中的key。以下两种接口与两种xml文件，均可以配合使用。但是推荐在接口上使用@Param注解，在xml文件中，也明确指定参数名称。

  1. 接口：

    List<PmsAttr> queryAttr(String attrId, String attrName);

    List<PmsAttr> queryAttr(@Param("attrId") String attrId, @Param("attrName") String attrName);

  2. xml文件

    1. 使用参数名称
    <select id="queryAttr" resultType="com.it.mybatis.bean.PmsAttr">
    select *
    from pms_attr
    where attr_id = #{attrId}
    and attr_name = #{attrName}
</select>
    1. 使用param1、param2
    <select id="queryAttr" resultType="com.it.mybatis.bean.PmsAttr">
    select *
    from pms_attr
    where attr_id = #{param1}
    and attr_name = #{param2}
</select

# 2.2 将多个参数封装在一个POJO中

在新增一条数据的时候，我们可以将所有的参数封装在一个POJO对象中。parameterType属性可以指定，也可以不指定。

例一：在接口中不实用注解，在xml文件中获取参数时，在#{}中直接写POJO中的参数值

  1. 接口

    void insertAttr(PmsAttr pmsAttr);

  2. xml文件

    <insert id="insertAttr"  parameterType="com.it.mybatis.bean.PmsAttr">
    insert into pms_attr (
        attr_id, attr_name, search_type, value_type, icon,
        value_select, attr_type, enable, category_id, show_desc)
    values (
        #{attrId}, #{attrName}, #{searchType}, #{valueType}, #{icon},
        #{valueSelect}, #{attrType}, #{enable}, #{categoryId}, #{showDesc}
    );
</insert>

例二：在接口中使用了注解

  1. 接口

    void insertAttr(@Param("pmsAttr") PmsAttr pmsAttr);

  2. xml文件

    <insert id="insertAttr"  parameterType="com.it.mybatis.bean.PmsAttr">
    insert into pms_attr (
        attr_id, attr_name, search_type, value_type,
        icon, value_select, attr_type, enable, 
        category_id, show_desc)
    values (
        #{pmsAttr.attrId}, #{pmsAttr.attrName}, #{pmsAttr.searchType}, #{pmsAttr.valueType}, 
        #{pmsAttr.icon}, #{pmsAttr.valueSelect}, #{pmsAttr.attrType}, #{pmsAttr.enable}, 
        #{pmsAttr.categoryId}, #{pmsAttr.showDesc}
    );
</insert>

  3. 注意：如果在接口中使用注解，而在xml文件中仍然直接获取POJO中的属性时，会出现以下异常：

    org.mybatis.spring.MyBatisSystemException: nested exception is org.apache.ibatis.binding.BindingException: Parameter 'attrId' not found. Available parameters are [pmsAttr, param1]

# 2.3 将多个参数封装在一个Map中

  1. 使用注解

    1. 接口

      List<PmsAttr> queryAttr(@Param("map") Map<String, Object> map);

    2. xml文件

      <select id="queryAttr" resultType="pmsAttr">
    select *
    from pms_attr
    where attr_id = #{map.attrId}
    and attr_name = #{map.attrName}
</select>

  2. 不使用注解

    1. 接口

      List<PmsAttr> queryAttr(Map<String, Object> map);

    2. xml文件

      <select id="queryAttr" resultType="pmsAttr">
    select *
    from pms_attr
    where attr_id = #{attrId}
    and attr_name = #{attrName}
</select>

  3. 注意

    • 如果使用了注解，则xml文件中，需要使用`map.`来获取值

    • 如果没有使用注解，则直接使用属性

# 2.4 编写一个POJO，专门用于数据查询

如果每次查询的参数有多个，每一次都编写Map，不太方便，可以直接编写一个POJO对象，专门用户数据查询传参。

# 2.5 其他情况

  1. 情况一

    List<PmsAttr> queryAttr(@Param("attrId") String attrId, String attrName)

    attr_id = #{attrId/param1} and attrName = #{param2}

  2. 情况二

    List<PmsAttr> queryAttr(String attrId, @Param("pmsAttr") PmsAttr pmsAttr)

    attr_id = #{param1} attr_name = #{param2.attrName/pmsAttr.attrName}

  3. 情况三

    传入的是集合或数组，MyBatis会讲集合或数组封装在Map中，Map的key可以分以下几种情况

    1. 集合(Collection、Set)：collection
    2. List：list、collection
    3. 数组：array
    List<PmsAttr> queryAttr(List<String> attrIds)

attr_id = #{list[0]}
attr_id = #{collection[0]}

    List<PmsAttr> queryAttr(@Param("ids") List<String> attrIds)
attr_id = #{attrIds[0]}

# 三、#{}与${}的区别

#与$都能够将参数中的值取出，拼接在SQL语句中。

区别：

  1. #{}：是以预编译的形式，将参数设置到SQL语句中，能够反之SQL注入
  2. ${}：取出的值直接拼接在SQL语句中，存在安全问题。

大多数情况下，都使用#{}

在原生SQL都不支持占位符的情况下，可以使用${}来完成操作，例如：

  1. 分表

    select * from ${year}_salary where XXX

  2. 排序

    select * from ${year}_salary order by ${field_name} ${order};
select * from 2020_salary order by salary desc;

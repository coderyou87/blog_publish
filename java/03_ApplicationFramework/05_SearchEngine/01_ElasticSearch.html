# ElasticSearch

# 一、ElasticSearch介绍

# 1.1 引言

  1. 在海量数据中执行搜索时，如果使用常规关系型数据库，效率很低
  2. 如果关键字输入不准确，一样可以搜索到想要的数据
  3. 将搜索的关键字，以红色的字体显示

# 1.2 ES介绍

ES是使用Java语言，并且基于Lucene编写的搜索引擎框架，提供了分布式的全文搜索功能，提供可一个基于RESTful风格的WEB接口，官方客户端也对多种语言提供了相应的API。

  • Lucene：是一个搜索引擎的底层
  • 分布式：ES主要是为了突出它的横向扩展能力
  • 全文检索：将一段文本内容进行分词，并且将分出的单个词语统一存储到一个分词库中。在搜索时，根据关键字在分词库中检索，查找相关的内容。（倒排索引）
  • RESTful风格的WEB接口：ES提供了一套基于HTTP请求的操作，根据请求方式与参数的不同，就能事项对应的功能。

# 1.3 ES的起源

# 1.4 ES与 Solor的区别

  1. Solor在查询静态数据时，效率高于ES；如果数据是实时变化的，Solor的查询效率会下降很多，但ES的查询效率基本不会发生变化。
  2. 搭建Solor集群的时候，需要依赖Zookeeper；搭建ES集群的时候，不需要第三方软件或依赖
  3. Solor针对国内的文档不多；ES国内社区非常活跃，文档健全
  4. ES对现在的云计算和大数据支持很好

# 1.5 倒排索引

​ 将存放的数据，以一定的方式进行分词，并且将分词的内容存放到一个单独的分词库中。当用于查询数据时，先对查询关键字进行分词，然后在分词库中查询关键字，得到数据的ID标识。根据ID标识，在数据库中获取对应的内容。

# 三、ElasticSearch基本操作

# 3.1 ES的结构

# 3.1.1 索引Index

ES的服务中，可以创建多个索引

每一个索引可以被分成5片存储

每一个分片至少会存在一个备份分片

备份分片默认不会提供检索服务，当ES检索压力过大时，才会提供服务，默认情况，只是保障数据安全

备份的分片需要放在不同的服务器中

# 3.1.2 类型 Type

一个索引下，可以创建多个Type，相当于MySQL中的库

根据ES的版本不同，类型的创建也不同

# 3.1.3 文档 Document

一个类型下可以创建多个文档，相当于MySQL中的表

# 3.1.4 属性 Field

一个文档中，可以有多个属性，相当于MySQL中的列

# 3.2 操作ES的RESTful语法

GET 请求：

​ 查询索引信息： http://ip:port/index

​ 查询指定文档信息：http://ip:port/index/type/doc_id

POST 请求：

​ 查询文档，可以在请求体中添加json字符串来代表查询条件：http://ip:port/index/type/_search

​ 修改文档，在请求体中添加json字符串来代表修改的具体信息：http://ip:port/index/type/doc_id/_update

PUT 请求：

​ 创建一个索引，需要在请求体中执行索引信息： http://ip:port/index

​ 创建索引时，指定创建的索引、类型、文档、属性等信息：http://ip:port/index/type/_mappings

DELETE 请求：

​ 删除索引以及相关的文档等全部信息：http://ip:port/index

​ 删除执行文档： http://ip:port/index/type/doc_id

# 3.4 ES中Field可以指定的类型

字符串：

  • text：一般用于全文检索，Field会被分词

  • keyword：关键词，不会被分词

数值类型：

  • long, integer, short, byte, double, float, half_float, scaled_float

时间类型：

  • date：可以指定具体的时间类型（yyyy-MM-dd HH:mm:ss||yyyy-MM-dd||epoch_millis）

布尔类型：

  • 表达true或false

二进制类型：

  • binary类型暂时支持Base64 编码的字符串

范围类型：

  • integer_range, float_range, long_range, double_range, date_range：赋值时，无需执行具体的内容，只需要存储一个范围即可（gt、lt、gte、lte）

经纬度类型：

  • geo_point：用来存储经纬度

IP类型：

  • ip：存储IPV4或者IPV6

# 6.2 match查询

match查询属于高级查询，会根据查询字段的类型，采用不同的查询方式：

  • 查询的字段类型是日期或数值，会将查询的类容转换为对应的类型
  • 查询的字段类型是keyword，match不会对查询的参数进行分词，直接进行内容匹配
  • 查询的字段类型是text（可以被分词），match会对查询的参数进行分词，然后在分词库中匹配

match查询，底层实际上是多个term查询，会将多个查询的结果封装后一并返回。

# 6.4 深分页scroll

ES对from+size是有限制的，from与size的和，不能超过10000

from+size在ES中的查询原理：

  1. 将用户传入的插叙参数分词
  2. 将分词得到的数据在分词库中检索，得到多个文档Id
  3. 根据文档ID，在各个分片中拉取数据（耗时较长）
  4. 根据score，对数据进行排序（耗时较长）
  5. 根据from与size的值，截取指定行数的数据
  6. 返回数据

Scroll+size在ES中的查询原理：

  1. 将用户传入的插叙参数分词
  2. 将分词得到的数据在分词库中检索，得到多个文档Id
  3. 将文档的Id存放在ES的一个上下文中（内存）
  4. 根据指定的size到ES中检索指定的数据，已经获取数据的文档ID，会从上下文中移除
  5. 如果需要下一页的数据，再从上下文中获取文档ID，然后获取数据
  6. 循环第四步与第五步

    searchSourceBuilder.size(4);
    searchSourceBuilder.sort("fee", SortOrder.DESC);
    searchSourceBuilder.query(QueryBuilders.matchAllQuery());

    SearchResponse response = client.search(searchRequest, RequestOptions.DEFAULT);

    String scrollId = response.getScrollId();
    System.out.println(scrollId);
    System.out.println("=============================首页===========================");
    // 获取首页数据
    for (SearchHit searchHit : response.getHits().getHits()) {
        System.out.println(searchHit.getSourceAsString());
    }

    // 获取其他页的数据
    while (true) {
        // 通过SearchScrollRequest去查询其他页的数据
        SearchScrollRequest searchScrollRequest = new SearchScrollRequest(scrollId);
        // 指定scrollId的生存时间
        searchScrollRequest.scroll(TimeValue.timeValueMinutes(5L));
        // 发送请求
        SearchResponse searchResponse = client.scroll(searchScrollRequest, RequestOptions.DEFAULT);
        SearchHits hits = searchResponse.getHits();
        // 如果没有数据，结束循环
        if (hits == null || hits.getHits().length == 0) {

        }
        SearchHit[] searchHits = hits.getHits();
        System.out.println("===========================下一页==========================");
        for (SearchHit searchHit : searchHits) {
            System.out.println(searchHit.getSourceAsString());
        }
    }

    ClearScrollRequest clearScrollRequest = new ClearScrollRequest();
    clearScrollRequest.addScrollId(scrollId);

# 6.6.2 boosting 查询

boosting查询可以帮助我们影响查询后的score

  • positive：只有匹配上positive的查询内容，才会被放到返回的数据中
  • negative：如果查询到的数据同时匹配上了positive和negative，就可以降低文档分数
  • negative_boost：指定系数，必须小于1.0

查询时的分数，如何计算：

  • 搜索的关键字在文档中出现的频次越高，分数越高
  • 指定的文档内容越短，分数越高
  • 查询参数被分词后，分词内容在分次库中匹配的个数越多，分数越高

        "match": {

        }
      },
      "negative": {
        "match": {
          "smsContent": "蚂蚁"
        }
      }, 
      "negative_boost": 0.5
    }
  }
}

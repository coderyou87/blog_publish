# Spring原理——PostProcessor扩展

Spring容器初始化的主要步骤：

  1. 加载保存和读取所有bean配置
  2. 按照之前的配置创建bean

# 一、BeanFactoryPostProcessor

BeanPostProcessor与BeanFactoryPostProcessor的区别：

  • BeanPostProcessor：bean后置处理器，在bean创建对象初始化前后进行拦截工作
  • BeanFactoryPostProcessor：beanFactory后置处理器，在BeanFactory标准初始化之后调用。此时，所有的bean定义已经保存并加载到BeanFactory，但是bean的实例还未创建

测试BeanFactoryPostProcessor的工作时间：

创建一个自定义的BeanFactoryPostProcessor，让其实现BeanFactoryPostProcessor接口，将该处理器加载到容器中，在postProcessBeanFactory方法中，获取相关的bean定义信息。

package com.it.extension;

import org.springframework.beans.BeansException;
import org.springframework.beans.factory.config.BeanFactoryPostProcessor;
import org.springframework.beans.factory.config.ConfigurableListableBeanFactory;
import org.springframework.stereotype.Component;

@Component
public class MyBeanFactoryPostProcessor implements BeanFactoryPostProcessor {

    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) throws BeansException {
        System.out.println("MyBeanFactoryPostProcessor postProcessBeanFactory");
        int beanDefinitionCount = beanFactory.getBeanDefinitionCount();
        System.out.println("beanDefinitionCount = " + beanDefinitionCount);
        String[] beanDefinitionNames = beanFactory.getBeanDefinitionNames();
        for (String beanDefinitionName : beanDefinitionNames) {
            System.out.println("beanDefinitionName = " + beanDefinitionName);
        }
    }
}

创建一个配置类，在其中加载自定义的组件

package com.it.extension;

import com.it.domain.Person;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;

@ComponentScan("com.it.extension")
@Configuration
public class ExtensionConfig {

    @Bean
    public Person person() {
        return new Person();
    }
}

# 二、BeanDefinitionRegistryPostProcessor

BeanDefinitionRegistryPostProcessor的执行时间：所有的Bean定义已经被加载，但是都还没有实例化，在BeanFactoryPostProcessor之后。BeanDefinitionRegistryPostProcessor是BeanFactoryPostProcessor的子接口。利用BeanDefinitionRegistryPostProcessor可以额外给容器中添加一些组件。

自定义BeanDefinitionRegistryPostProcessor：

package com.it.extension;

import com.it.domain.Person;
import org.springframework.beans.BeansException;
import org.springframework.beans.factory.config.ConfigurableListableBeanFactory;
import org.springframework.beans.factory.support.BeanDefinitionRegistry;
import org.springframework.beans.factory.support.BeanDefinitionRegistryPostProcessor;
import org.springframework.beans.factory.support.RootBeanDefinition;
import org.springframework.stereotype.Component;

@Component
public class MyBeanDefinitionRegistryPostProcessor implements BeanDefinitionRegistryPostProcessor {

    /**
     * @param registry Bean定义信息的保存中心，保存了所有Bean的定义信息，
     *                 BeanFactory就是按照registry中保存的bean定义信息来进行实例化的
     * @throws BeansException
     */
    @Override
    public void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) throws BeansException {
        System.out.println("MyBeanDefinitionRegistryPostProcessor  postProcessBeanDefinitionRegistry");
        System.out.println(registry.getBeanDefinitionCount());
        System.out.println("====================================");
        //BeanDefinitionBuilder beanDefinitionBuilder = BeanDefinitionBuilder.rootBeanDefinition(Person.class);
        registry.registerBeanDefinition("hello_person", new RootBeanDefinition(Person.class));
    }

    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) throws BeansException {
        System.out.println("MyBeanDefinitionRegistryPostProcessor  postProcessBeanFactory");
        System.out.println(beanFactory.getBeanDefinitionCount());
        System.out.println("+++++++++++++++++++++++++++++++++++++");
    }
}

postProcessBeanDefinitionRegistry()方法比postProcessBeanFactory()方法先执行。

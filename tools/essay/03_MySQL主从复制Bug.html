# MySQL主从复制Bug

# Slave_IO_Running: NO

在MySQL主从复制时，Slave_IO_Running为No，需要将其修改为Yes。

  1. 使用reply-log避免从机hostname改变

    [Warning] Neither --relay-log nor --relay-log-index were used; so replication may break when this MySQL server acts as a slave and has his hostname changed!! Please use '--relay-log=localhost-relay-bin' to avoid this problem.

    在/etc/my.cnf文件中追加一下内容

    relay-log=localhost-relay-bin

  2. MySQL启动时的server UUIDS相同

    因为从机是从主机克隆出来的，在克隆之前，MySQL就已经安装完毕，所以在从机启动后，使用的server UUIDs与主机是相同的，而在MySQL主从复制时，主从机的UUIDs不能相同。

    [ERROR] Slave I/O for channel '': Fatal error: The slave I/O thread stops because master and slave have equal MySQL server UUIDs; these UUIDs must be different for replication to work. Error_code: 1593

    修改MYSQL_HOME/data/auto.cnf中的server-uuid或直接删除auto.cnf文件

# Slave_SQL_Running: No

操作从库：

stop slave;
SET GLOBAL SQL_SLAVE_SKIP_COUNTER=1; 
START SLAVE;
show slave status\G

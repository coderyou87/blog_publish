# 学习RocketMQ过程中遇到的bug

# 启动RocketMQ，出现 [2]+Exit 253

我的原因是：在配置RocketMQ时，指定了其存储数据的位置，但是创建的文件夹过多，所以出现了这个bug。在搭建集群时，需要创建的文件夹有：

mkdir /usr/local/rocketmq/store
mkdir /usr/local/rocketmq/store/commitlog
mkdir /usr/local/rocketmq/store/consumequeue
mkdir /usr/local/rocketmq/store/index

配置文件按照以下格式书写

#存储路径
storePathRootDir=/usr/local/rocketmq/store
#commitLog 存储路径
storePathCommitLog=/usr/local/rocketmq/store/commitlog
#消费队列存储路径存储路径
storePathConsumeQueue=/usr/local/rocketmq/store/consumequeue
#消息索引存储路径
storePathIndex=/usr/local/rocketmq/store/index
#checkpoint 文件存储路径
storeCheckpoint=/usr/local/rocketmq/store/checkpoint
#abort 文件存储路径
abortFile=/usr/local/rocketmq/store/abort

# 在同一台机器上，使用2主2从方式启动RocketMQ，启动第一个master没出现问题，启动第二个时出现bug：java.lang.RuntimeException: Lock failed,MQ already started

因为在同一台机器上会启动两个broker，所以需要有两个存储路径，在各自的配置文件中分别指名。如果使用了同一个存储路径，就会出现以上错误。

# 在RocketMQ消费者方面，使用SQL进行过滤时，出现的bug：CODE: 1 DESC: The broker does not support consumer to filter message by SQL92

这是RocketMQ不支持SQL过滤，在对应的broker配置文件中添加以下配置，并重启broker即可

enablePropertyFilter=true
